# 출력 패턴

스킬이 일관되고 고품질의 출력을 생성해야 할 때 이 패턴을 사용.

## 템플릿 패턴

출력 형식을 위한 템플릿 제공. 요구사항에 따라 엄격함 수준을 조정.

**엄격한 요구사항 (API 응답이나 데이터 형식 등):**

```markdown
## 보고서 구조

반드시 이 정확한 템플릿 구조를 사용할 것:

# [분석 제목]

## 요약

[주요 발견사항의 한 단락 개요]

## 주요 발견사항

- 뒷받침 데이터가 있는 발견사항 1
- 뒷받침 데이터가 있는 발견사항 2
- 뒷받침 데이터가 있는 발견사항 3

## 권고사항

1. 구체적이고 실행 가능한 권고
2. 구체적이고 실행 가능한 권고
```

**유연한 가이드 (적응이 유용할 때):**

```markdown
## 보고서 구조

합리적인 기본 형식이지만, 최선의 판단을 사용할 것:

# [분석 제목]

## 요약

[개요]

## 주요 발견사항

[발견한 내용에 따라 섹션 조정]

## 권고사항

[특정 맥락에 맞게 조정]

분석 유형에 따라 섹션을 필요에 맞게 조정.
```

## 예시 패턴

출력 품질이 예시를 보는 것에 좌우되는 스킬의 경우, 입력/출력 쌍을 제공:

```markdown
## 커밋 메시지 형식

다음 예시를 따라 커밋 메시지 생성:

**예시 1:**
입력: JWT 토큰으로 사용자 인증 추가
출력:
```

feat(auth): JWT 기반 인증 구현

로그인 엔드포인트와 토큰 검증 미들웨어 추가

```

**예시 2:**
입력: 보고서에서 날짜가 잘못 표시되는 버그 수정
출력:
```

fix(reports): 타임존 변환 시 날짜 형식 수정

보고서 생성 전반에서 UTC 타임스탬프를 일관되게 사용

```

이 스타일을 따를 것: type(scope): 간략한 설명, 이후 상세 설명.
```

예시는 설명만으로는 전달하기 어려운 원하는 스타일과 세부 수준을 Claude가 이해하는 데 도움을 준다.
