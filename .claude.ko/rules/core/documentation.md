# 문서화 정책

## 주석은 기술 부채 - 적극적으로 최소화

### 허용 (AI + Human 최적화)

#### WHY

- 비즈니스 규칙
- 외부 제약사항
- 법적 요구사항
- 직관에 반하는 결정

#### Constraints (AI 위반 방지)

- 성능: `// Constraint: 100ms 내 완료 필수`
- API 제한: `// Constraint: 배치당 최대 100개`
- 멱등성: `// Constraint: 재시도 안전성을 위해 멱등성 필수`

#### Intent (AI 최적화 방향 안내)

- `// Goal: DB 왕복 최소화`
- `// Goal: 성능보다 가독성 우선`

#### Side Effects (AI 부작용 인지)

- `// Side effect: 이메일 발송`
- `// Side effect: 캐시 무효화`

#### Anti-Pattern (AI 리팩토링 방지)

- `// 의도적으로 순차 실행 - 병렬화 시 멱등성 깨짐`
- `// 의도적으로 장황함 - 디버깅 최적화, 간결성 아님`

### 금지

- 코드 흐름 설명
- WHAT 설명
- 이름 보완
- 섹션 구분자
- 주석 처리된 코드
- 타입 정보 (TypeScript가 처리)

### 기본 자세

"이 주석을 더 나은 코드로 제거할 수 있는가?" → 90%의 경우 가능

코드가 WHAT을 설명하는 주석이 필요하면, 주석 대신 코드를 고쳐라 (이름 변경, 함수 추출).

### AI 컨텍스트 고려사항

전략적 WHY/Constraint/Intent 주석은 AI가 코드 분석으로 동일한 정보를 추론하는 데 필요한 토큰보다 적은 비용이 든다. 최대 효과를 위해 관련 코드 바로 앞에 주석 배치.
