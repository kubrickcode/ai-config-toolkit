# Issue Documentation Guide

## 기본 원칙

- 간결하고 명확하게
- 실행 가능한 예시와 함께
- "왜"에 집중
- 문서는 기본적으로 번역을 부탁하지 않는 이상 항상 한국어로 작성할 것
